- Viral ratio = Views / Channel Subscribers

- Views per day = Total Views / Days since upload
- Estimated watch time = Views × Average % viewed (estimate from retention curve if available)

**Qualitative Analysis**:
- Thumbnail style
- Title strategy
- Hook (first 15 seconds)
- Structure/pacing
- Visual style
- Voice/narration style
- Call-to-action

### 2. Title Analysis

**Title Patterns**:

| Pattern | Example | Viral Ratio | When to Use |
|---------|---------|-------------|-------------|
| **Question** | "AIエージェントとは？" | ___ | Creates curiosity |
| **How-to** | "AI自動化の作り方" | ___ | Educational content |
| **Shock/Surprise** | "驚愕！AIが開発を完全自動化" | ___ | Attention-grabbing (use sparingly) |
| **Number** | "AI開発の5つの革命" | ___ | Clear value proposition |
| **Comparison** | "人間 vs AI、どっちが速い？" | ___ | Creates interest |
| **Benefit-focused** | "AI で開発速度が10倍に" | ___ | Clear value |

**Title Formula Analysis**:

Most successful titles follow:
```
[Hook] + [Topic] + [Benefit/Result]

Examples:
"[世界初！] [自律型AIエージェント] [が開発を完全自動化]"
"[驚きの性能] [Flickering Scenery] [でコーディング不要に]"
```

**Character Count**:
- Japanese: 25-35 characters (visible in search/suggestions)
- English: 50-60 characters

**Keyword Placement**:
- Primary keyword in first 20 characters
- Power words: 世界初、驚き、革命、完全、自動、簡単、わかりやすく

### 3. Hook Analysis (First 15 Seconds)

**Analyze opening of each video**:

| Hook Type | Example | Effectiveness | When to Use |
|-----------|---------|---------------|-------------|
| **Problem Statement** | "開発が遅すぎる問題、解決します" | High | Problem-solution videos |
| **Shock/Statistics** | "AIエージェントが人間の100倍速" | High | Data-driven content |
| **Question** | "なぜAIは自分でコードを書けないのか？" | Medium | Educational content |
| **Preview Result** | "これを見てください（デモ）" | Very High | Demo-based content |
| **Story/Anecdote** | "ある日、開発が完全に自動化されました" | Medium | Storytelling format |

**Key Elements of Strong Hooks**:
1. Visual: Show something interesting in first 3 seconds
2. Audio: Strong opening statement
3. Text: On-screen text reinforcing hook
4. Promise: What viewer will learn/gain
5. Urgency: Why watch now

### 4. Video Structure Analysis

**Common Structures**:

**Structure A: Problem-Solution** (Best for our use case)
```
1. Hook (0:00-0:15) - "Development is too slow"
2. Problem (0:15-2:00) - Explain current challenges
3. Solution Intro (2:00-3:00) - Introduce Flickering Scenery
4. How It Works (3:00-7:00) - Technical explanation
5. Results (7:00-9:00) - Benchmarks, demos
6. Call-to-Action (9:00-10:00) - GitHub, paper, support
```

**Structure B: Story-Driven**
```
1. Hook (0:00-0:15) - Relatable scenario
2. Journey (0:15-3:00) - How we got here
3. Discovery (3:00-5:00) - The breakthrough
4. Explanation (5:00-8:00) - How it works
5. Impact (8:00-9:30) - What it means
6. CTA (9:30-10:00)
```

**Structure C: Tutorial**
```
1. Hook (0:00-0:15) - What you'll learn
2. Prerequisites (0:15-1:00) - What you need to know
3. Step 1 (1:00-3:00)
4. Step 2 (3:00-5:00)
5. Step 3 (5:00-7:00)
6. Results (7:00-9:00)
7. Next Steps (9:00-10:00)
```

**For Flickering Scenery**: Use **Structure A (Problem-Solution)**

### 5. Visual Style Analysis

**Categories**:

| Style | Examples | Pros | Cons | Viral Potential |
|-------|----------|------|------|-----------------|
| **Yukkuri Characters** | Traditional ゆっくり解説 | Familiar, low production cost | May feel dated | High in Japanese market |
| **Live Action** | Presenter on camera | Personal connection | Requires equipment | Medium |
| **Screencasts** | Code/software demos | Shows actual product | Can be boring | Medium |
| **Animated** | Motion graphics | Professional, engaging | High production cost | High |
| **Hybrid** | Mix of above | Best of all worlds | Complex production | Very High |

**For Flickering Scenery**: Use **Hybrid** (Yukkuri + Screencasts + Diagrams)

### 6. Pacing Analysis

**Successful Videos**:

| Segment | Time per Screen/Concept | Pacing Style |
|---------|------------------------|--------------|
| Intro | 3-5 seconds | Fast (grab attention) |
| Problem | 10-15 seconds per point | Medium (build tension) |
| Solution | 20-30 seconds per feature | Medium (explain clearly) |
| Technical | 30-45 seconds per concept | Slow (let it sink in) |
| Demo | 15-20 seconds per demo | Fast (show, don't tell) |
| Results | 10-15 seconds per metric | Fast (build excitement) |
| CTA | 20-30 seconds | Medium (clear instructions) |

**Average Screen Changes**: Every 5-7 seconds (keeps attention)

## Red Flags to Avoid

**From Competitor Analysis**:

| Red Flag | Why It Fails | How to Avoid |
|----------|--------------|--------------|
| **Clickbait titles** | Lose trust, high bounce | Be honest, deliver on promise |
| **Slow intro** | Viewers leave in first 15s | Hook immediately |
| **Too technical** | General audience lost | Simplify, use analogies |
| **No clear value** | Why should I care? | State benefit clearly |
| **Poor audio** | Unwatchable | Use VOICEVOX quality settings |
| **No CTA** | Viewers don't know what to do next | Clear GitHub/paper links |
| **Too long** | Viewer fatigue | Keep to 8-12 minutes |
