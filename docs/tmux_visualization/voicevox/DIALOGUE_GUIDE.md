## 台本フォーマット

### 基本構造

```text
REIMU: セリフ内容
MARISA: セリフ内容
REIMU: セリフ内容
```

### メタデータ（オプション）

```text
# EVENT: phase_theta1_start
# DURATION: 15s
# EMOTION: excited

REIMU: ねぇねぇ魔理沙、何か始まったよ！
MARISA: おっ、θ₁の理解フェーズだぜ。いよいよ本番だな
```

### 特殊指定

```text
REIMU: ねぇねぇ{{1.0}}これって何なの？  # {{1.0}} = 1秒の間
MARISA: [excited]これはすごいぜ！      # [excited] = 興奮した声
REIMU: <volume:1.2>すごーい！</volume>   # 音量調整
```

## キャラクター設定詳細

### 霊夢 (Reimu) - おバカ聞き手キャラ

**基本性格**:
- 少し抜けている、天然ボケ
- 難しいことは苦手だけど素直
- 視聴者の代弁者として質問する
- 魔理沙の説明に感心する

**口調の特徴**:
- 「〜なの？」「〜だよね？」（疑問形が多い）
- 「へぇ〜」「すごい！」「わぁ！」（感嘆詞）
- 「よくわかんないけど〜」「難しいなぁ」（素直な反応）
- 語尾は柔らかい女性口調

**NGワード**:
- 専門用語の使用（魔理沙に質問する側）
- 冷静すぎる分析
- 長すぎる説明

**セリフ例**:
```
良い例:
✅ 「ねぇねぇ、これって何やってるの？」
✅ 「へぇ〜、そうなんだ！すごいね！」
✅ 「よくわかんないけど、うまくいってる感じ？」

悪い例:
❌ 「θ₁フェーズのアルゴリズムは効率的ですね」
❌ 「このコンバージェンス率は理論値と一致しています」
```

### 魔理沙 (Marisa) - 知識解説キャラ

**基本性格**:
- 頭が良く技術に詳しい
- でも難しい言葉は使わず、わかりやすく説明
- 成功すると一緒に喜ぶ
- ちょっと自信家だけど優しい

**口調の特徴**:
- 「〜だぜ」「〜なんだ」（男の子っぽい語尾）
- 「要するに〜ってことだ」（わかりやすく言い換え）
- 「さすがだな！」「いいぞ！」（褒める）
- 一人称は「私」または省略

**NGワード**:
- 女性的すぎる語尾（「〜わ」「〜よ」）
- 上から目線の説明
- 霊夢をバカにする発言

**セリフ例**:
```
良い例:
✅ 「今はθ₁の理解フェーズだぜ」
✅ 「要するに、ユーザーの意図を読み取ってるんだ」
✅ 「さすがだな！完璧な動作だぜ」

悪い例:
❌ 「今はθ₁の理解フェーズですわ」
❌ 「こんな簡単なことも分からないの？」
```

**台本A（基本版）**:

**台本B（興奮版）**:

## 台本作成のコツ

### 1. テンポを意識する

**良い例**:
```text
REIMU: ねぇ魔理沙
MARISA: なんだ？
REIMU: これすごくない？
MARISA: ああ、すごいぜ
```

**悪い例**:
```text
REIMU: ねぇ魔理沙、ちょっと聞いてもいいかな、これって本当にすごいんじゃないかなって思うんだけど
MARISA: そうだね、確かにこれは素晴らしい実装だと思うよ、技術的にも高度だし
```

### 2. キャラクターの役割分担

- **霊夢**: 質問、驚き、感嘆
- **魔理沙**: 回答、解説、評価

### 3. 感情を込める

```text
# 喜び
REIMU: やった〜！すごいよ！

# 驚き
REIMU: えっ！？本当に！？

# 心配
REIMU: 大丈夫かな…ちょっと不安…

# 感動
REIMU: なんか…すごく感動しちゃった…
```

### 4. 間を使う

```text
REIMU: これって{{0.5}}すごいの？
MARISA: ああ{{1.0}}超すごいぜ
```

### 5. ゆっくり解説風に

- 視聴者が理解できるペース
- 専門用語は魔理沙が噛み砕く
- 霊夢の疑問 = 視聴者の疑問

## カスタム台本の追加

### 1. 台本ファイル作成

```bash
cd /Users/shunsuke/Dev/AI_Agent/docs/tmux_visualization/voicevox/dialogues

# カテゴリ選択（または新規作成）
mkdir -p custom

# 台本ファイル作成
vim custom/my_custom_event.txt
```

### 2. 台本記述

```text
# EVENT: my_custom_event
# DURATION: 10s
# EMOTION: happy

REIMU: わぁ！カスタムイベントだ！
MARISA: 自分で作った台本が動いてるぜ
REIMU: すごい！簡単にできちゃうんだね！
MARISA: まぁな、これがシステムの柔軟性ってやつだぜ
```

### 3. イベント登録

```bash
# config/narration_events.json に追加
vim ../config/narration_events.json
```

```json
{
  "custom_events": [
    {
      "event_name": "my_custom_event",
      "dialogue_file": "custom/my_custom_event.txt",
      "priority": "normal",
      "enabled": true
    }
  ]
}
```

### 4. テスト実行

```bash
./scripts/voicevox_narrator.sh event my_custom_event
```
